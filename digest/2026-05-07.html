Two frontier models trained without a single NVIDIA GPU shipped this week: DeepSeek V4 on Chinese silicon, ZAYA1-8B on AMD. Anthropic's Compute Bet Ships With Agent Primitives

Anthropic gained access to SpaceX's Colossus 1 in Memphis (300+ MW, 220K+ GPUs) and doubled Claude Code and API rate limits today. This is the fifth compute deal, after Amazon, Google, Broadcom, and Microsoft. The compute matters because Anthropic simultaneously shipped three agent primitives: Dreaming (agents autonomously review past sessions and surface recurring mistakes), Outcomes (rubric-driven self-grading where a separate grader scores agent output in isolated context, looping up to 20 iterations until passing), and Multiagent Orchestration (sub-agents on a shared filesystem, traceable in Console). Dreaming is the bet that autonomous self-reflection beats hand-tuned prompts for long-running workflows.

The Access vs. Meaning Split Decides Which Agents Ship

Every agent demo this year shows a model reaching something new: a browser, a form, a payment page. Access and understanding look identical in a demo but diverge in production. The agent clicks "refund" without knowing if the customer is entitled or a human should approve. Coding agents arrived first because code exposes meaning natively: tests, diffs, CI failures tell the agent it's on track without constant supervision. The product test for every announcement: does this give the model access, or meaning?

Videos

How I AI — "Claude Code Just Got WAY More Powerful"

Claire Vo recaps five practical announcements from Anthropic's Code with Claude developer event: Routines (cron + HTTP/GitHub-webhook scheduling, runs locally or in cloud), Outcomes (rubric-driven self-grading agent loops, up to 20 iterations), Multi-Agent orchestration (up to 25 agents in shared container with explicit hierarchy), Dreams (on-demand cross-session memory consolidation), and doubled usage limits across Pro/Max/Team. Frames Anthropic's strategy as "platform, not product flash" — shipping primitives rather than mind-blowing features.

Key takeaways:

  • The unit of work shifts from "prompt" to "evaluation criteria" — write a rubric for what good looks like and let the agent iterate against it
  • Multi-agent teams are now an API primitive, not prompt convention — define orchestrator/delegate hierarchy programmatically
  • Dreams is "what we do when we dream" reframed as a deliberate primitive (review N past sessions, decide what's worth committing)
  • Agent forgetting is the missing primitive — memory accumulates infinitely, nobody is solving the purge/decay side
  • No single announcement is mind-blowing; together they signal Anthropic wants to own the agent-builder substrate

Greg Isenberg × Mang — "Google's design.md is a design team in a file"

Designer Mang demos Google's open-source design.md spec — a markdown blueprint that captures a design system's "soul" (typography, colors, spacing, WebGL/3D animation rules) and travels with prompts across Lovable, V0, Aura, Stitch, Codex, Claude Code. He frames it as the design equivalent of agents.md / skills.md, demonstrates remixing across mediums (web → mobile → motion → slides), and argues taste plus niche specificity are the only durable moats left.

Key takeaways:

  • HTML is the "finished dish," design.md is the "recipe," skills are the "ingredients" — portable design DNA persists across screens, mediums, and platforms
  • Don't dump everything into agents.md — it burns tokens and doesn't apply per-workflow. Split: design.md per project, skills per workflow, leave agents.md minimal
  • Baseline design quality is now extremely high — "purple gradient" used to be wow, now it's a red flag. Lasers, WebGL, skeuomorphic textures, niche-specific signatures are what make people click
  • "You cannot compete with people who are generalized" — taste is the moat; the practice is using every app in your niche, following every maker, building a second brain for design inspiration
  • AI makes you work more, not less — Mang admits 1,000-10,000 prompts per product, building four products solo. The "care" Steve Jobs talked about now lives in the prompting workflow

This Week in AI ep 12 — "Is Anthropic a Cult? AI Beats ER Doctors & Recursive Self-Improvement"

Three-way roundtable (Jason Calacanis + Naveen + Trey Halterman) spanning AI in healthcare, recursive self-improvement, hyperscaler capex, and frontier-lab cult dynamics.

Key takeaways:

  • AI ER diagnosis already beats physicians — Harvard study: o1 hit 67% correct diagnosis vs 50-55% for two ER docs across 76 real Beth Israel cases; perfect clinical reasoning on 98%. Liability and insurance complexity, not technology, is the real bottleneck.
  • The "context problem" still blocks deployment — getting the right context (body language, patient history, partial info) into the model is the hard part. ER works on incomplete context.
  • Open-source EHR is the Epic disruption play — Epic = 80% of records, 40% margin, "duct taped together," $100-500M to implement. Combine open-source EHR + AI scribes + design data for LLM consumption (recency vs relevancy) instead of human consumption.
  • Recursive self-improvement is already partially happening — Dario: "Claude designs the next version of Claude itself, not completely but most of it. Engineers at Anthropic say they don't write any code anymore." Naveen takes the OVER on Jack Clark's 60% probability for end-of-2028; predicts "really good" recursive self-improvement around 2030.
  • Hyperscaler capex going parabolic — Morgan Stanley raised forecast to $805B in 2026, $1.1T in 2027, carried by only 5 hyperscalers. Naveen: "we're reshaping the economy."
