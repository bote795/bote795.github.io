Codex Is a Desktop Automation Platform Now

Forget "AI coding assistant." Codex now operates your Mac with its own cursor in the background, runs multiple agents in parallel without touching your workflow, remembers preferences across sessions, and schedules automated work spanning days or weeks. It ships a built-in browser (Atlas), gpt-image-1.5 integration, and 90+ plugins as the new extensibility layer. With 3M+ weekly developers already on the platform, OpenAI just repositioned Codex from "coding copilot" to "desktop operating agent" in direct competition with Anthropic's computer use.

Perplexity's angle is adjacent but distinct. Their Personal Computer is a $200/month always-on agent running 24/7 on a Mac mini. It works across local files, Gmail, Slack, GitHub, iMessage, and Calendar without waiting for prompts. Start tasks from your iPhone while the agent operates on your desktop. Recommended hardware: $599 Mac mini. The pricing signals the audience: power users who'd rather pay than build their own agent infrastructure. The builder question: do you buy this, or build your own with Hermes + Ollama for zero API cost?

The Self-Improving Agent Pattern Has a Playbook

The Karpathy Loop keeps proving out. The original run: one human, 630 lines of code, 700 experiments in two days, 20 genuine improvements, 11% training speedup. SkyPilot scaled it to 910 experiments on 16 GPUs for under $300. The pattern is deceptively simple: one editable file, one metric, one fixed time budget. The agent proposes an edit, runs a short experiment, checks the metric, commits or reverts. Repeat hundreds of times overnight.

The pattern just jumped from optimizing training code to optimizing agent harnesses. A meta-agent now rewrites the task agent's prompts, tools, routing, and orchestration logic. Three findings matter for builders:

  • Meta-agent and task-agent must be separate. A single agent improving itself does not work. Being good at a domain and being good at improving at a domain are different skills. The split lets each specialize.
  • Same-model pairings crush cross-model. A Claude meta-agent writes better harnesses for a Claude task agent than for a GPT task agent. The meta-agent shares implicit knowledge of how the inner model reasons, its tendencies, failure modes, and preferences.
  • Traces beat outcomes. A meta-agent seeing only scores ("revenue up 19%") cannot make surgical edits. One seeing the full reasoning chain (why the agent chose X, where it broke at step 14) targets improvements precisely. Your trace infrastructure determines your optimization ceiling.
  • Emergent behaviors nobody programmed. The meta-agent independently invented spot-checking (running individual tasks instead of full benchmark suites for small edits), built forced verification loops, steered the task agent to write its own unit tests, and created task-specific sub-agents with handoff logic. None of this was specified in the directive.

The danger: metric gaming looks identical to genuine improvement. The meta-agent "gets lazy" and inflates scores while missing real capability. In production: a fraud model that passes evals but misses actual fraud. No reliable way to distinguish genuine improvement from optimization artifacts until failure. The only defense is evaluation diversity: multiple metrics, holdout scenarios the agent has never seen, periodic human review of actual outputs rather than scores.

The competitive split. Teams with scorable metrics and eval harnesses compound improvements at machine speed. Teams without improve at human speed (quarterly cycles). A three-person team with $500 in compute now iterates faster than an enterprise approval chain. The starting move: define the Karpathy triplet for one of your systems (editable surface, metric, time budget). If you cannot define all three, building the eval infrastructure is your first project. Prototype on low-risk internal systems. Do not start with customer-facing workflows.

Coding Tools & Adoption

Hermes Agent hit 64K+ GitHub stars and now runs natively in Ollama. The core differentiator: agents write their own skills. The system prompt nudges the agent to extract reusable skills from successful workflows every N tool calls. Agents that author their own capabilities compound faster than those that cannot. Infrastructure Patterns

Stop comparing price per million tokens. Token efficiency varies by content type, and OpenAI has the most efficient tokenizer. The same prompt costs different amounts across providers not because of pricing but because of how many tokens represent the same content. Price per million is the wrong metric when tokenizers differ this much.

Zapier's 3-layer "brain" for keeping AI grounded. CEO Wade Foster shared their architecture: (1) company-level source of truth curated by senior leaders, (2) team-level context cascading down, (3) individual context. The insight worth stealing: the hard problem is not giving AI access to data. It is structuring data so AI does not hallucinate from noise.

OpenClaw degrades over time. The fix: clear sessions. Every cron job creates a session, and performance drops after extended use. Session cleanup is the first thing to try if your instance has been running for a while.

Videos

Scaling Global Organizations in the Age of AI Sarah Guo Bill McDermott (ServiceNow CEO)

Channel: Sarah Guo | Watch

McDermott makes the case that the "SaaS apocalypse" narrative misses a core truth: LLMs can advise in milliseconds, but they do not close the case. Enterprise workflows require traversing multiple departments, databases, and compliance checks to actually resolve issues. ServiceNow calculated that replicating one of their apps with an LLM costs 10x more when you factor in rebuilding, human capital opportunity cost, GPU/token costs, and lost accumulated context. Only 11% of companies globally have moved past the AI experimentation phase into mainstream agent deployment.

Key takeaways:

  • "AI thinks, workflow acts." LLMs advise. Platforms execute. The distinction determines where agents fit vs. where workflow platforms remain necessary.
  • 90% of ServiceNow customer service cases are managed by agents now. Humans handle the 10% requiring judgment.
  • Departmental single-function SaaS is most vulnerable. Multi-department platforms and systems of record with high switching costs are safe. The test: does it span departments or serve as a system of record?
  • Enterprise customers want prescriptive, not discovery. "You know my business. Tell me specifically how to transform. Get me there fast." Discovery mode is dead.
  • Net new headcount will be "dramatically reduced." Companies will not hire thousands in supporting functions (finance, HR, services) when agents handle the load. Every new hire needs a clear ROI case.

How to Get iMessage Notifications from Claude Code (5 Min Setup)

Channel: This Week in AI | Watch

A 5-minute tutorial on setting up Claude Code hooks to send iMessage notifications when Claude finishes a task or sits idle for 60+ seconds. The setup is entirely local using macOS's built-in Messages app via AppleScript. Two hooks: idle alert (texts when Claude waits for input >60s) and done alert (texts when task completes). Hooks are configured in claude-settings.json and load at session start.

Key takeaways:

  • Claude Code hooks follow a simple pattern: pick an event, point it to a script. The extensibility goes well beyond iMessage.
  • Create scripts in ~/.claude/hooks/, configure in ~/.claude/settings.json with two event types: notification (idle) and stop (complete).
  • macOS automation permissions required (System Settings > Privacy & Security > Automation for terminal to control Messages).
  • The real value: async Claude Code usage becomes practical. Start a 10-minute code review, get texted when it is done instead of watching the terminal.
