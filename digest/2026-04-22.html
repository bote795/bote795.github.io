Claude Code leaked 512K lines of its own TypeScript source via a misconfigured npm package, shipped a security flaw that bypasses developer-configured deny rules, and briefly appeared at $100/month on Anthropic's pricing page. All in the same week it hit #1 on Code, Document, and Vision Arenas. Separately, the most consequential architectural question in AI tooling right now is not which model to use. It is whether your knowledge system does the hard thinking when information comes in or when you ask a question.

Deep Dives

The architecture decision most builders are making by accident

Every AI knowledge system has one foundational fork: does the AI synthesize when information arrives, or when you query it?

Andrej Karpathy's viral wiki is a write-time system. When a new source arrives, the AI reads it, extracts what matters, updates topic pages, flags contradictions, and builds cross-references. The thinking happens once. Karpathy describes it as "compiled once, kept current, not re-derived on every query." His setup: Obsidian for browsing, an LLM for writing, the wiki as the codebase.

Most RAG systems, ChatGPT file uploads, and NotebookLM are query-time systems. The AI re-derives everything fresh on every question. Your hundredth query costs exactly as much as your first. Nothing compounds.

Where the wiki wins: Deep research where each new paper integrates against prior synthesis. Solo use. Domains where connections between sources matter more than individual facts. It also survives without AI running it's readable text files.

Where the wiki cracks: Team use. Two agents editing the same page create semantic conflicts. Fast-moving domains re-synthesizing on every status update is expensive. Precise queries: "show me every meeting where pricing was discussed" requires a database. The dangerous failure mode is staleness. A neglected database looks like a gap. You know you're missing something. A neglected wiki reads with confidence and is wrong. You don't question it because it sounds like knowledge you already have.

One subtlety to build into any team system: sometimes contradictions are the signal. A system that resolves "engineering says 12 weeks, sales promised 8" into coherent prose hides the most important strategic tension in the room. A database that stores both without resolving them preserves it.

The hybrid that solves both: Keep a structured database as your permanent store. Everything goes in, tagged and queryable. Run a wiki compiler on a schedule that reads the database and produces synthesized pages. The database is always the source of truth. The wiki is the compiled, regenerable view on top. Fix an error in the underlying data, regenerate the page. The wiki never drifts because it's always rebuilt from reality. Typed reasoning edges (supports / contradicts / supersedes) between entries make the synthesis richer than either system alone.

What to build right now:

  • Solo, one research topic, want it running fast: paste Karpathy's gist into Claude Code it builds the system for you.
  • Multi-agent access, high volume, structured queries: database-first (OB1 repo).
  • Want both: OB1 with the wiki compiler recipe, which is live and schedulable.

One deeper principle both approaches share: the AI is a maintainer, not an oracle. Its job is sustained, compounding work on an artifact not one-off answers. That division of labor (human judgment, AI maintenance) is what makes knowledge compound instead of accumulate.

Claude Code: three incidents in one week

1. Source code leak. A misconfigured source map in an npm package update exposed approximately 512K lines of internal TypeScript source. No customer data was exposed. Researcher Chaofan Shou flagged it. The code was implementation detail, not credentials but it's a gap in hardening for a tool deployed in sensitive codebases.

2. Security flaw enabling bypass of developer-configured deny rules. High severity. If you have Claude Code deployed with security policies, check for patches now. Malicious actors can silently evade user-configured rules. This is the more serious of the two.

3. Pricing confusion. Anthropic silently updated claude.com/pricing to show $100/month with no announcement, then reverted. Claude Code access was also removed from the $20/month Pro tier with no official communication. The tool is shipping fast and catching up on hardening in public. If you assumed the security posture or pricing was stable, it isn't.
