Builder's Daily May 15, 2026

Anthropic passed OpenAI in business adoption the same week both labs launched competing billion-dollar enterprise deployment ventures. Claude Code rate limits doubled because a SpaceX compute deal added 220,000 new Nvidia GPUs overnight. Both data points say the same thing: model access is the commodity now, and deployment is where the value gets captured.

The Real AI War: Who Owns the Deployment Layer

On May 4, Anthropic and OpenAI each announced PE-backed enterprise deployment companies. Anthropic's: a reported $1.5B vehicle backed by Blackstone, Hellman & Friedman, and Goldman Sachs, targeting mid-market companies with enough process complexity to need Claude but not enough internal AI engineering to deploy it. OpenAI's: a $10B vehicle anchored by TPG, with OpenAI guaranteeing PE backers a 17.5% annual return over 5 years. That fixed-yield instrument converts equity upside into a bond-like product which tells you how badly OpenAI wants this distribution channel. (OpenAI also acquired consulting firm Tomoro a week later, bringing ~150 engineers directly into the deployment vehicle.)

The diagnosis behind both moves: model access is fully commoditized. The implementation layer is the new moat. It has six specific components: workflow design, data access, authority (what actions the model can take and against which systems), evaluation, audit trails, and recovery/ownership. Most companies have built two. The companies that have built all six are starting to pull away from everyone else.

What the gap looks like in practice: a customer support AI that saves some writing time is not a deployment. A real deployment specifies which tickets get auto-classified, which customer records the model reads, which policy rules govern refunds, when a manager must approve, and which metrics prove it worked. The distance between those two things is where the enterprise value sits, still uncaptured.

For startups: generic "AI for enterprise productivity" now faces pressure from four directions at once. Labs moving down-stack with vertical templates and forward-deployed teams. Consultancies (McKinsey, BCG, Accenture, Capgemini) moving up with agentic build practices. Systems of record (Salesforce, SAP, Workday, ServiceNow) exposing agent APIs with native permissions and native audit trails. PE becoming a distribution channel that can push one deployment partner across an entire portfolio. Win condition: own a specific workflow, a vertical, or a control layer or build the infrastructure that lets companies evaluate agent output against real business outcomes.

Weekly market roundtable covering Anthropic's $200B Google compute commitment, its compute deal with Elon's xAI/SpaceX, the Cerebras IPO, and the public-market reckoning happening across SaaS (HubSpot, Cloudflare, Zoom Info, Monday). The throughline: capital is consolidating into a Google/Anthropic/OpenAI oligopoly; legacy SaaS gets either an AI-acceleration story or a terminal-decay valuation; intensity, not effort, separates billion-dollar founders; and any growth-at-1% public SaaS company is now PE bait.

Key takeaways:

  • Anthropic locked down secondary sales — but it's a nothing-burger: New rule requires board approval for all secondaries/SPVs. Standard practice for 5+ years per Jason; Anthropic told Menlo last year to stop running SPVs and people kept going anyway. They named specific bad-actor hedge funds. Mindset shift: Before: secondary markets were a free-for-all. After: tighter cap-table control as IPOs approach.
  • Anthropic-xAI/SpaceX compute deal signals consolidation, not friendship: Grok had an 11%-utilized data center. xAI is effectively switching from net-buyer of compute to net-seller. The deal pulls xAI from "$250B competitor" to "$25B notionally profitable supplier." Lesson: have lunch with competitors twice a year; you never know when the enemy becomes the customer.
  • Google's $200B Anthropic commit = ~40% of Google's total future backlog: The hyperscalers are deeply dependent on two private AI labs. Google is happy to lose at the model layer if they capture margin via TPUs + GCP. Microsoft is in a worse spot — same dependency on OpenAI but without a competitive model of their own.
  • Goldman's 24x token-consumption forecast by 2030 is too low: Jason argues parallel agents push the number to 250x+. Mindset shift: Before: sequential LLM use, one task at a time. After: 10 parallel agents producing 10 versions of a feature, LLM judge picks the best 2-3, human approves.
  • Token waste is real — only S-tier engineers should get $20K/mo budgets: A growing CTO backlash: mediocre "web devs" burn massive tokens to produce 100K-line blogs. Goodhart's Law warning — once you measure token usage, employees distort. The 100x engineers earn the budget; the rest are token-trashers.

Hermes Agent Just Made OpenClaw Look Old (@ThisWeekinAIPodcast)

Watch

A walkthrough of Hermes Agent, a local-first open-source autonomous AI agent from "New Research" that the host argues has dethroned Open Claw. Key advantages: 87 built-in skills, 80+ tool integrations, a polished dashboard with analytics/logs/secrets management, multi-agent profiles with per-agent soul.md system prompts, and an auto-triaged Kanban board driven by a cron job.

Key takeaways:

  • Local-first agents are the new ceiling: Hermes runs on the laptop, uses the user's API keys, and stores everything locally — talking via terminal, Slack, or Telegram. Mindset shift: Before: agents are cloud services you log into. After: the agent lives on your machine and the dashboard is for inspection/config, not execution.
  • Auto-generated skills from conversation: Hermes captures memory and auto-generates new skills based on what you do with it — meaning the agent's capability set grows organically rather than requiring manual skill authoring.
  • soul.md per-profile system prompts: Each multi-agent profile has its own personality file (soul.md) defining role, voice, examples, and rules. The host has 7 profiles: default workhorse, designer, researcher, reviewer, runner (Haiku for cost), and a domain-specific "This Week in AI X writer" with voice samples and post format examples baked in.
  • Model-per-task assignment: Different profiles use different models — Kimi K2.6 as default workhorse, Opus 4.7 for design, Haiku for cheap runner tasks. The right model for the job rather than one model for everything.
  • Lead with planning questions, not commands: Both for Hermes and Open Claw, the host recommends opening with "if I were to do this, how would we set it up?" to let the agent plan and think deeply before executing.
